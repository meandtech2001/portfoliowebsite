What this solution offers
End-to-end patient lifecycle on Dynamics 365
🏥
Patient Lifecycle BPF
Multi-stage Business Process Flow — Admission → Lab → Billing → Discharge — with branching logic and mandatory data validation at each stage.
⚙️
Custom API & Actions
Reusable server-side Custom APIs and Custom Actions for billing automation and appointment logic — callable from JS and Power Automate without code duplication.
☁️
Azure Integration Layer
Asynchronous event-driven messaging via Azure Service Bus + Logic Apps. Azure Functions handle lightweight data transformation between D365 and external systems.
📂
SharePoint Document Hub
SharePoint libraries connected to D365 patient records — automatic folder creation per patient with structured medical document management.
🧩
Virtual Tables & PCF
External healthcare data surfaced directly in D365 via Virtual Tables. Custom PCF controls and Custom Pages deliver a modern, role-specific UI.
🔄
Hybrid Automation Stack
Classic Workflows for synchronous instant execution combined with Power Automate for external integrations — full coverage of the D365 automation spectrum.
Solution Workflow
How the system flows
1
👤
Patient Registration
New patient record created in D365. BPF activates. Docs auto-linked to SharePoint.
2
🧪
Lab & Treatment
Lab orders triggered via Custom Actions. Azure Service Bus relays results from external lab systems.
3
💳
Billing Automation
Custom API calculates billing. Power Automate triggers invoice generation and payment alerts.
4
✅
Discharge & Reporting
Final stage closes BPF. Dashboards reflect real-time patient and billing status.
